Directory Structure Overview

Will we could give you a list of every file that is included with this package, we think it would be redundant as currently it may change to rapidly to actually be worth having. Once the project tends to stable out we may reconsider.

Instead we are opting to show a general overview of the directories, this should allow you to become familiar enough to understandorganization concept of StageHands.

stagehands(root directory)

This directory will contain all the files for the project. Loose files such as the README, CHANGELOG, LICENSE, etc... will bestored here as well.

src

Short for 'source' this is the directory were developers can create their own versions of the project software. It contains the pre-complied source code.

This directory also contains the following sub-directories;

SCSS

This directory contains all the SCSS style code for the project.

This directory also contains the following sub-directories;

components

This directory contains all the style code for StageHands class components.

data

This directory contains all the data variables needed for the SCSS compilation.

This directory also contains the following sub-subdirectories;

Vendor

The default data variables that come with the project. Broken up into maps and lists.

functions

This directory contains all the SCSS functions used throughout the project.

Mixins

This directory contains all the SCSS mixins used throughout the project.

normalize

This directory contains all the files related to github.com/necolas/normalize.css

placeholders

This directory contains all the files related to the SCSS placeholders used throughout the project.

styles

This directory contains all the default styles that get applied to elements. It contains sub-directories where each element is placed according to there element category. See; Mozilla Developer: Element References.

test_styles

This directory contains the files that are used to style the document pages individually. Things like the example boxes,sectioning, etc. This directory is NOT related or included in the main library.

tests

This directory contains a test file used to unsure the SCSS functions are working properly. See;Testing Function

prod

Short for 'production' and is the resulting compiled code from 'src' This code will be used in the production environment.

This directory will also contain some directories form 'src', but not all as some files are only used during SCSS compilation.

fonts

Directory to store the projects fonts

tests

Directory contains the project documentation, the documentation also serves as a test for the project itself thus its name.

We hope by now your are beginning to recognize that StageHands is very specific about where it stores files. The reasoning behind this is to achieve two things

  1. Intuitive feeling. By having this level of specificity we hope to create a sort of intuitive feeling when working with the project.
  2. Modularity. When you have your project broken up this much it is very easy to be precises about what pieces of the project you wantto take with you.
